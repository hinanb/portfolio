## ACID Properties

- **Atomicity** — Transactions are all-or-nothing
- **Consistency** — Database moves from one valid state to another
- **Isolation** — Concurrent transactions don't interfere
- **Durability** — Committed data survives system failures

## Database Types

| Type | Use Case | Examples |
|------|----------|----------|
| Relational | Structured data, transactions | PostgreSQL, MySQL, Oracle |
| Key-Value | Caching, sessions | Redis |
| Document | Flexible schemas | MongoDB |
| Column-Family | Time-series, analytics | Cassandra |

## Indexing

Indexes speed up reads but slow down writes. Choose indexes based on query patterns, not every column.

## Normalization vs Denormalization

Normalize for write consistency; denormalize for read performance when query patterns are well understood.
